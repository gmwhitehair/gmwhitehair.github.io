  • By breaking our code into smaller parts it makes our code easier to work with and easier to find problems within. Also, we can copy and paste certain parts of the code if they are similar like in the Rock, Papers, Scissors code.
  • We constantly check and test our code to check for errors. If we continue writing code with those errors and mistakes, it is going to be hard to diagnose and fix the problem when we are finished writing the code. It will be even worse if we are copying that code with the errors. Also, if we test the code, we can easily diagnose where the error is located.
